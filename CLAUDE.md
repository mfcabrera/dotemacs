# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Personal Doom Emacs configuration for macOS. The active config lives in `.doom.d/` — the `oldconf/` directory is a legacy Prelude-based config and should not be modified.

## Key Files

- `.doom.d/init.el` — Doom module declarations (what's enabled). After changing, run `doom sync`.
- `.doom.d/packages.el` — Extra packages beyond Doom defaults. After changing, run `doom sync`.
- `.doom.d/config.el` — Main configuration (~1080 lines). All `use-package!`, `after!`, keybindings, and custom functions.
- `.doom.d/bigui.el` — Personal utility library (`bigui/` prefixed functions), loaded via `(load! "bigui.el")` in config.el.
- `.doom.d/custom.el` — Auto-generated by Customize; generally don't edit directly.
- `.doom.d/snippets/` — YASnippet templates for org-mode and python-mode.

## Commands

```bash
# After changing init.el or packages.el:
doom sync

# Full rebuild:
doom sync -u

# Check for issues:
doom doctor
```

## Architecture Notes

**No evil-mode** — This config does NOT use vim keybindings. Evil is commented out in init.el. Uses default Emacs keybindings with `+smartparens`.

**Platform abstraction** — The `mac-or-linux` macro in config.el handles platform-specific values (fonts, paths). Currently macOS-only in practice.

**Org-mode is central** — Heavy org-mode setup with:
- org-roam (v2) for knowledge management, directory set to `~/PersonalDrive/org-notes/`
- org-journal (monthly files in `archive/journal/`)
- org-super-agenda for grouped agenda views
- Custom agenda commands: `w` (work), `h` (home/personal), `D` (daily), `W`/`F` (weekly reviews)
- org-ref + org-roam-bibtex for bibliography management
- Capture templates for todos, work items, reading backlog, blog ideas

**Org file structure** — Agenda files are gathered recursively from `inbox/` and `projects/` and `areas/` subdirs under org-directory.

**LLM integration** — gptel configured with custom directives, plus mcp.el for MCP servers (filesystem, memory, Google Calendar, Linear). llm-tool-collection provides additional gptel tools.

**Claude Code IDE integration** — `claude-code-ide.el` with custom MCP tools exposed to Claude Code:
- Agenda views (home/work/daily)
- Org buffer revert, org-roam DB sync, org-agenda refresh
- Org-roam search, file opening
- Anki note pushing via AnkiConnect

**Python development** — Python with pyright LSP, conda (miniconda at `/opt/homebrew/Caskroom/miniconda/base`), python-black, py-isort, flycheck with flake8+mypy chain.

**Secrets** — Uses 1Password CLI (`op read`) via `bigui/read-1pw-secret` for database credentials and similar secrets. Never hardcode credentials.

## Conventions

- Personal utility functions use the `bigui/` prefix (defined in bigui.el).
- Local helper functions use `my/` prefix (defined inline in config.el, primarily MCP tools).
- Package config uses Doom's `use-package!` and `after!` macros, not vanilla `use-package`.
- The `cloud-drive-dir` variable points to `~/PersonalDrive/` on macOS and is used for bibliography, org-images, etc.
